Platform Power Changelog

Version 1.5.2

Changes:
  • Remove thrown menu prompts from the world.
  • Add an error message for when not all of the teams are ready and someone tries to start the game.
  • Change from a chat message to a title that the game is starting.
  • Add a control menu item for force-starting the game regardless of pre-game checks. This should rarely be used, but can be used if people on teams are being annoying and not ready-ing up after a while.


Bug Fixes:
  • Fix preventing the game from occasionally starting due to weird behaviour with the team enable/disable functions.
  • Fixes edge case if all players died at the same time (or the game was only started with no one on an actual team) it will no longer hang in a perma-end state.
  • Adds a tag to the fireballs that get summoned to prevent them from flickering in and out of existance as that was super annoying.

Version 1.5.1

Changes in version 1.5.1:
  • Add the recommended server properties to the archive.

Version 1.5.0

Changes in version 1.5.0:
  • Added support for Minecraft 1.15.2
  • Made the game controls much cleaner by using an inventory menu rather than a command book. This prevents players from needing to add the op tag to themselves and allows a more seamless experience for everyone. The menu includes a prompt for the hotbar that displays the keybind the user has the open inventory key set to.
  • Added team randomization options.
  • Spectator is now not the default team. Unless the game has started, in which case everyone without a team will be put onto the spectator team.
  • To restrict menu control to those who are event organizers you can run /tag @s add control as the user who wants control. This tag can be given to as many players as desired. This also exposes an extra setting to remove everyone from all teams.

Version 1.4.0

Changes in version 1.4.0:
  • Changed player detection from team based to effect based. This prevents players leaving in the middle of a game from stopping the game from finishing. This means that everyone in the game will always have the minecraft:unluck effect, this does not impact gameplay at all, it is just a method to be able to see all players in the game for each team.
  • Updated the list of blocks that the game uses to clear the map, this should prevent any rogue blocks from slipping through the reset system that is in place. The added tags are minecraft:block_list and minecraft:wall_signs (overwritting default)
  • After the game you will be teleported back to the zone in the lobby for the team you were on. (Ex: Someone on the green team will be teleported back to the green circle in the lobby). This makes it easier to play with the same teams multiple times in a row if desired.
  • Added a debug function, can give you information by running /function classic:debug, this will give you important information for when you are requesting help or reporting a bug.

Version 1.3.0

Changes in version 1.3.0:
  • Spectators now see blocks.
  • Spectators now get put into spectator mode when the game starts rather than survival.

Version 1.2.1

Changes in version 1.2.1:
  • Added a visual indicator of what the button does in the middle of the lobby.
  • Updates README.txt to include how to get the control book and start the game.
